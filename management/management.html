Governance

Partitions

This section will discuss the logical partitions of the City Data Model Collaborative Environment. The major partitions are expected to include:

  • Collaborative Elements: Elements of the environment that are developed, approved, and maintained collaboratively under the auspices of the W3C according to the governance structure defined by this document.
  • Submitted Elements: Elements of the environment that have been contributed by other entities as input that should be considered when developing, approving, and maintaining the Collaborative Elements. Submitted elements to the environment often only provide partial specifications with the full content provided as a reference to an external document, which might fall under a more restrictive copyright.

It is envisioned that all elements of the City Data Model, except for the Physical Data Model elements, will be collaborative elements. Further, it is envisioned that all elements within the Physical Data Model will be submitted elements; however, it is expected that the Physical Data Model will include submittals from multiple sources, which will need to be distinguished from each other and managed in a hierarchical tree. For example, it is likely that all ISO submittals will use a standard copyright statement, but when considering a submittal we will need to consider which technical committee and standard it represents.

Maturity Levels of Elements

The City Data Model will define terms, classes, properties, associations, etc. and the model will evolve over time. Many diagrams contained within the model will need to depict multiple model elements and not all of the model elements will necessarily be at the same level of maturity. Each element (e.g., term, class, property) defined within the model will mature according to its own development cycle and this maturity status will be recorded in the element's metadata. Framework

The framework specification contained in this specification conforms to ISO 42010 [[ISO42010]]. The viewpoint specifications are based on the Viewpoint Specification Template [[Hilliard]]. The Information Viewpoint specification was developed in consideration of the specifications defined in Software Systems Architecture [[Rozanski]], the Department of Defense Architecture Framework [[DoDAF]], The Open Group Architecture Framework [[TOGAF]], and the Zachman Framework [[Zachman]].
